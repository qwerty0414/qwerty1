# デモモード - スタンドアロン実行ガイド

このアプリケーションはデモモードで、外部サーバーやセットアップなしにローカル環境で完全に動作します。

##  クイックスタート

### 1. ブラウザを開いて下記にアクセス
\\\
http://localhost:8000
\\
`

### 2. ログイン画面で、以下のいずれかのデモアカウントを使用

| ロール | Email | パスワード | 権限 |
|--------|-------|-----------|------|
| ホールスタッフ | \hall@demo.local\ | \password\ | テーブル注文管理 |
| キッチンスタッフ | \kitchen@demo.local\ | \password\ | 調理指示表示 |
| 管理者 | \dmin@demo.local\ | \password\ | 全機能 |

### 3. ブラウザを更新するとデータがリセット

メモリ内DBを使用しているため、ブラウザの更新（F5）でアプリケーションが再起動され、新しいデモデータが自動生成されます。

##  デモデータ

起動時に自動生成されるデータ：

- **テーブル（席）**: 10卓（1-01 〜 1-10）
- **従業員**: ホールスタッフ、キッチンスタッフ、管理者 各1名
- **既存注文**: 3卓に注文確定済み
- **コールリクエスト**: 2件のサンプル要求

##  特徴

 **追加セットアップ不要**
- PHPの別途インストール不要
- MySQL/PostgreSQLなどの外部DBサーバー不要
- .env 設定ファイルの編集不要

 **完全にメモリ内で動作**
- ローカルマシンのみで実行
- インターネット接続不要
- クライアント企業への展示に最適

 **リセット可能**
- ブラウザ更新で完全にリセット
- テストや デモンストレーション時に便利
- 各セッションで新鮮なデータから開始

##  手動でのデモモード有効化

localhost 以外からのアクセスでもデモモードを有効化したい場合、\.env\ ファイルに以下を追加：

\\\env
DEMO_MODE=true
\\\

すると、どのアドレスからアクセスしてもデモモードで動作します。

##  ログ確認

デモモード初期化のログを確認するには：

\\\ash
tail -f storage/logs/laravel.log | grep -E 'Demo mode|[]'
\\\

##  問題解決

### "Database cannot be created" エラー

**原因**: PHP がインストールされていない

**解決**: 以下からPHPをインストール
- Windows: https://windows.php.net/ または Laravel Herd
- macOS: \rew install php\
- Linux: \sudo apt-get install php php-sqlite3\

### デモアカウントでログインできない

**解決**: ブラウザを更新してデータを再生成

### メモリDB が複数プロセスで共有されない

これは仕様です。各アクセスはメモリ内の独立したDBを使用します。本番運用時はOS別の方法で固定DBを使用してください。

##  複数ユーザーでのテスト

デモモードは開発デモンストレーション用です。複数ユーザーの並行操作をテストしたい場合は、別のDB接続を使用する設定に変更してください。
